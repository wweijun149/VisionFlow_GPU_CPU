# AOI CV Based 流水帳報告

日期：2026-07-02（星期四）至 2026-07-08（星期三）

本週主要工作集中在 VisionFlow AOI 的 Monitor 長時間運行穩定性、Batch/Monitor 記憶體使用量控制、監控畫面刷新效能，以及目前專案狀態文件整理。整體方向是讓系統在連續處理大量影像時不要累積過多明細資料，也讓 GUI 在監控模式下更新更順。

## 2026-07-02（星期四）

- 先處理 Batch 與 Monitor 長時間執行時的記憶體保留問題。
- 新增 `core/result_compactor.py`，把檢測結果壓縮成 GUI 需要的精簡資料。
- Batch inspection 完成單張影像後，不再把完整 pipeline 結果全部留在記憶體中，只保留摘要、輸出路徑、tile 基本資訊、detector pass/score 與 defect 必要欄位。
- Monitor folder inspection 也改成使用精簡結果，完整細節仍保留在報告 JSON，GUI 端只拿必要資料做表格與 scatter 顯示。
- 在 Batch 與 Monitor 的單張處理流程結束後清掉暫存 result，並觸發輕量 GC，降低連續跑圖時的記憶體壓力。
- 調整 Monitor 的檔案穩定性追蹤資料，影像排入佇列後清掉 `_file_states` 中的對應紀錄，避免長時間監控資料夾時持續累積檔案狀態。
- 優化 Monitor 畫面刷新方式。原本每新增一筆結果會整個表格重刷，改成新資料直接插入第一列。
- Monitor 歷史紀錄限制為 200 筆，避免 GUI 表格無限制成長。
- Monitor cumulative sequence scatter 限制只取最近 50 筆資料，降低畫面重繪負擔。
- 新增表格列更新 helper，讓 Monitor table 的單列更新與整表初始化邏輯分開。
- 新增資料後會同步更新統計、表格、sequence scatter、選取列與單張 scatter，讓監控模式即時顯示更順。
- 調整 Monitor sequence scatter 的點大小，不再依 defect count 放大，避免累積圖上點位過大導致視覺混亂。
- 更新 scatter chart widget，讓點半徑縮放可以由呼叫端控制。
- 補強 README，整理目前 VisionFlow AOI 的功能狀態、架構、GUI 模式、輸出內容與後續方向，方便交接與回顧。

## 本週重點整理

- Monitor 長時間運行更穩定：限制 GUI 歷史資料與 scatter 資料量，降低長時間監控的重繪與記憶體壓力。
- Batch/Monitor 結果更輕量：GUI 只保留摘要，完整結果交給 JSON/report 保存。
- Monitor 畫面刷新更有效率：新增結果改為插入單列，不再每次整表重建。
- Scatter 視覺更可控：sequence scatter 不再因 defect count 放大點位，累積圖更容易閱讀。
- 文件狀態更完整：README 補上目前 VisionFlow AOI 的架構、功能與使用情境。
