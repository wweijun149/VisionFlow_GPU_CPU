# AOI CV Based 流水帳報告

日期：2026-07-09（星期四）至 2026-07-15（星期三）

本週主要工作集中在 VisionFlow AOI 的 CUDA GPU 加速架構。從前期規劃開始，陸續完成可由 recipe 控制的 GPU backend、CPU fallback、CUDA DLL 編譯與驗證工具、效能觀測、核心影像前處理優化、persistent context，以及通用 `PreprocessPlan`。同時也整理專案 roadmap 與開發規範，讓後續 detector、GUI、CUDA、測試及發佈工作都依同一份清單執行。

## 2026-07-09（星期四）至 2026-07-12（星期日）

- 本期間 repository 沒有新的 commit。
- 本週可核對的程式與文件變更集中在 7 月 13 日及 7 月 14 日。

## 2026-07-13（星期一）

- 建立 GPU 加速 roadmap，整理 AOI 從 CPU reference、CUDA 導入到 RTX 3090 實機驗收的工作方向。
- 新增獨立的 `cuda_practice/` 練習區，包含 device info、grayscale、threshold、Gaussian blur、morphology、template matching 與 ROI batch 等範例。
- 加入 `sm_86` 編譯腳本與說明，作為 RTX 3090 開發、編譯及基礎驗證的準備。
- 練習程式與正式 AOI pipeline 分開，降低實驗性 CUDA 程式影響產品主流程的風險。

## 2026-07-14（星期二）

- 新增 recipe 控制的 GPU backend。Recipe Designer 可分別設定切圖、GUI 預覽與各 detector 是否使用 GPU，且所有 GPU 開關預設關閉。
- 加入 `GpuRuntime` 與 CUDA DLL bridge，讓 pipeline、tiler、detector、GUI worker 與預覽顯示可以共用一致的 GPU 路由與狀態資訊。
- 補上安全 CPU fallback：DLL 不存在、CUDA device 不可用、初始化或執行失敗時，可依 recipe 設定回退 CPU，並在結果中記錄實際 backend 與原因。
- 建立可攜式 CUDA DLL 開發套件，包含公開 C ABI header、錯誤碼、內部 helper、PowerShell build script、C++ smoke test 與 Python 驗證工具。
- 補強 DLL build 與驗證流程，可檢查 ABI、primitive 結果、AOI CPU/GPU 等價性及 4K benchmark；正式 RTX 3090 編譯與執行驗證仍待目標機完成。
- 加入 GPU 效能觀測基線，記錄 pipeline 各階段、Reporter 輸出、DLL 載入、鎖等待、H2D/D2H bytes、round trips 與 primitive fallback 等資訊。
- 優化 CUDA Gaussian blur，改用 horizontal/vertical separable kernels 與 constant memory weights。
- 優化 Adaptive Mean Threshold，改用 replicate-border 的 64-bit integral image，並加入 structured validation cases。
- 為 Detector401-2 加入 persistent CUDA context、grow-only buffers、context stats 與 fused preprocessing，減少重複配置與不必要的中間傳輸。
- 建立 backend-neutral 的 `PreprocessPlan`、typed operators、`CpuPreprocessExecutor` 與 `CudaPreprocessExecutor`，並先將 Detector401-2 遷移到新架構。
- 將 CPU、GPU、CUDA、GUI、CI、打包及 RTX 3090 驗收項目合併到唯一的 `Todo.md`，移除重複的 GPU/CUDA Todo 文件。
- 更新 `AGENT.md`，明確定義模組責任、CPU/GPU 等價與 fallback 原則、測試要求、安全 staging、commit 與 push 流程。
- 建立 AOI 專案專用 workflow skills，統一一般修改、detector 開發、CUDA 驗證及 release 的執行方式。

## 2026-07-15（星期三）

- 彙整 2026-07-09 至 2026-07-15 的 Git 紀錄與 roadmap 完成項目，產出本週流水帳報告。
- 確認 `main` 已與 `origin/main` 同步；週報建立前的程式與文件工作均已提交。
- 目前正式 CUDA DLL 的 RTX 3090 `sm_86` 重建、primitive matrix、五份 production recipe CPU/GPU 等價、benchmark、VRAM 壓力及打包版驗收仍列為待辦，不能視為本週已完成的實機驗證。
- 工作目錄中的 Windows release ZIP 與 `outputs/logs/` 為未追蹤產物，本次不納入提交。

## 本週重點整理

- GPU 架構正式進入主流程：recipe、pipeline、detector 與 GUI 已具備一致的 CUDA 開關、狀態與 fallback 行為。
- CPU-only 仍是完整支援模式與正確性基準；GPU 功能採可選 DLL，失敗時不應破壞既有檢測流程。
- CUDA 開發工具更完整：公開 C ABI、build script、native smoke、Python validation 與 benchmark 入口已建立。
- 核心前處理完成第一輪效能優化：separable Gaussian、64-bit integral Adaptive Mean、persistent context 與 fused Detector401-2 preprocessing。
- detector 架構開始由 detector-specific CUDA workflow 轉向通用 `PreprocessPlan` 與 typed operators，方便後續擴充及維持 CPU/CUDA 語意一致。
- 效能追蹤範圍擴大到 pipeline、Reporter、資料傳輸、鎖等待與 fallback，可作為後續 RTX 3090 benchmark 的 baseline。
- 專案管理方式統一：只保留一份 `Todo.md`，並補齊架構規範、驗證矩陣及發佈流程。
